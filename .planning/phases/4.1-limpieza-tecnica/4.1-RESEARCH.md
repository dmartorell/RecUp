La API en `attachment.js` devuelve `uploaded: results` donde `results` es un array (línea 54). La comprobación correcta es `data.uploaded.length > 0`.

**Primary recommendation:** Ejecutar exactamente dos cambios — `git rm src/js/visualizer.js` y editar línea 220 de ticket-modal.js. Sin más.

## Architecture Patterns

### Patrón de Eliminación Segura de Dead Code

**Qué verificar antes de eliminar un archivo:**
1. ¿Hay imports del archivo en otros módulos? — `grep -r "visualizer" src/` → sin resultados en archivos `.js` activos
2. ¿Se referencia en HTML con `<script src>`? — No, el proyecto usa ES modules con `import`
3. ¿Hay referencias en tests? — No hay test suite automatizada en este proyecto

**Resultado verificado:** `visualizer.js` no aparece en ningún import de `app.js` ni de ningún otro módulo activo. El archivo es seguro para eliminar.

### Corrección del Bug de Coerción

**Contexto del bug (confirmado en código):**

En `attachment.js` (server), la respuesta de error incluye:
```js
// línea 54 de server/routes/attachment.js
uploaded: results,  // results es un Array
```

En `ticket-modal.js` (cliente), la detección:
```js
// línea 220 de src/js/ticket-modal.js — INCORRECTO
if (!res.ok) throw { partial: data.uploaded > 0, data };
```

**Por qué es un bug:**
- `[] > 0` → JS convierte `[]` a `""` → a `0` → `0 > 0` = `false` (correcto por casualidad)
- `[item] > 0` → JS convierte `[item]` a `"[object Object]"` → a `NaN` → `NaN > 0` = `false` (INCORRECTO — debería ser `true`)
- El comportamiento correcto requiere `.length > 0`

**Corrección:**
```js
// línea 220 de src/js/ticket-modal.js — CORRECTO
if (!res.ok) throw { partial: data.uploaded.length > 0, data };
```

## Common Pitfalls

### Pitfall 1: Eliminar con `rm` en lugar de `git rm`
**What goes wrong:** El archivo desaparece del filesystem pero git lo marca como "deleted" (unstaged). Hay que hacer `git add` adicional para stagear la eliminación.
**How to avoid:** Usar `git rm src/js/visualizer.js` directamente — elimina del filesystem Y stagea la eliminación en un solo comando.

### Pitfall 2: Olvidar verificar imports antes de borrar
**What goes wrong:** Si existiera algún import de visualizer.js que se pasó por alto, el servidor fallaría al cargar el módulo.
**How to avoid:** Verificación ya realizada — grep no encuentra ningún import activo. Confirmar antes de ejecutar.

### Pitfall 3: Asumir que `data.uploaded` podría ser `undefined`
**What goes wrong:** Si `data.uploaded` es `undefined` (por respuesta de error inesperada), `.length` lanzaría `TypeError`.
**How to avoid:** La API siempre devuelve `uploaded: results` en el path de error (líneas 51-55 de attachment.js). La propiedad siempre existe cuando `!res.ok`. No es necesario guard adicional, pero documentar la asunción.
