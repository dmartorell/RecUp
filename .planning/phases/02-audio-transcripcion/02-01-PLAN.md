  <action>
Crear los tres modulos JS independientes que manejan las APIs del navegador. Todos usan ES Modules (export/import).

**src/js/recorder.js** — MediaRecorder wrapper:
- `requestMicAccess()`: llama `navigator.mediaDevices.getUserMedia({ audio: true })`, retorna el stream. Si falla con NotAllowedError, lanza error descriptivo. Si falla con NotFoundError, lanza error descriptivo. Guardar el stream en variable de modulo para reutilizar.
- `getStream()`: retorna el stream guardado (para que visualizer y otros lo usen).
- `startRecording()`: crea MediaRecorder con mimeType `audio/webm;codecs=opus` (fallback a `audio/webm` si opus no soportado). Limpia chunks array. Llama `mediaRecorder.start()`. Configura `ondataavailable` para acumular chunks (solo si `e.data.size > 0`).
- `stopRecording()`: retorna Promise que resuelve con el Blob de audio. Llama `mediaRecorder.stop()`. En `onstop`, crea Blob con chunks y resuelve la promise.
- Exportar: `{ requestMicAccess, getStream, startRecording, stopRecording }`

**src/js/transcriber.js** — SpeechRecognition wrapper con auto-reinicio:
- Variable de modulo: `finalTranscript = ''`, `isActive = false`, `recognition = null`
- `initRecognition()`: crea instancia SpeechRecognition (con fallback webkit), configura: `continuous: true`, `interimResults: true`, `lang: 'es-ES'`. Configura handlers:
  - `onresult`: iterar desde `event.resultIndex`, acumular solo resultados `isFinal` en `finalTranscript`. Importante: NO resetear finalTranscript dentro del handler.
  - `onend`: si `isActive`, hacer `setTimeout(() => recognition.start(), 300)` (delay de 300ms para evitar rate-limiting, ver Pitfall 2 en RESEARCH.md).
  - `onerror`: si error es `no-speech` o `aborted` y `isActive`, ignorar (onend reiniciara). Para otros errores, llamar callback de error pasado como parametro.
- `startTranscription(onError)`: resetea `finalTranscript = ''`, `isActive = true`. Si recognition no existe, llama initRecognition(). Llama `recognition.start()`. Guarda onError callback.
- `stopTranscription()`: `isActive = false`, `recognition.stop()`. Retorna `finalTranscript`.
- `getTranscript()`: retorna `finalTranscript` actual.
- Exportar: `{ startTranscription, stopTranscription, getTranscript }`

**src/js/visualizer.js** — Waveform con AnalyserNode:
- `initVisualizer(stream)`: crea AudioContext, createMediaStreamSource del stream, createAnalyser (fftSize 256). Conecta source -> analyser (NO conectar a destination para evitar feedback). Guarda analyser y audioCtx en variables de modulo.
- `startVisualization(barsContainer)`: recibe el contenedor DOM con las barras. Inicia loop con `requestAnimationFrame`. En cada frame: `analyser.getByteFrequencyData(dataArray)`, mapear subset de datos a barras (12 barras, distribuidas por frecuencia). Actualizar `style.height` de cada barra como porcentaje (minimo 4% para que siempre se vea algo). Guardar animationFrameId para cancelar.
- `stopVisualization()`: cancela `requestAnimationFrame`. Resetea barras a altura minima. Llama `audioCtx.close()` para liberar recursos. Resetea variables de modulo.
- Importante: NO crear AudioContext al cargar la pagina (Pitfall 3). Se crea en initVisualizer que se llama tras click del usuario.
- Exportar: `{ initVisualizer, startVisualization, stopVisualization }`
