  <files>src/index.html, src/js/app.js, src/css/styles.css</files>
  <action>
Crear la interfaz completa de BugShot:

1. Crear src/index.html:
   - DOCTYPE html, lang="es"
   - Meta viewport para responsive
   - Google Fonts: Inter (weights 400, 500, 600, 700) con preconnect
   - Tailwind v4 CDN: `<script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>`
   - Bloque `<style type="text/tailwindcss">` con @theme:
     - --color-accent: #2563eb
     - --color-accent-hover: #1d4ed8
     - --font-sans: 'Inter', sans-serif
   - Link a css/styles.css
   - Script src="js/app.js" con type="module" al final del body

   - Estructura del body:
     a) `<div id="unsupported">` - Pantalla bloqueante de navegador incompatible:
        - DEBE usar CSS INLINE (no Tailwind) porque Tailwind podria no funcionar en navegadores incompatibles
        - position:fixed, inset:0, fondo blanco, centrado flex
        - Titulo "Navegador no compatible" (font-size 1.5rem, font-weight 600)
        - Texto: "BugShot necesita APIs que solo estan disponibles en navegadores basados en Chromium. Por favor, usa Chrome o Edge."
        - Color texto secundario: #666
        - Boton/enlace "Descargar Chrome" apuntando a https://www.google.com/chrome/
        - Estilo boton: bg #2563eb, color blanco, padding 0.75rem 1.5rem, border-radius 8px
        - display:none por defecto (app.js lo muestra si es incompatible)

     b) `<div id="app">` - App principal (visible por defecto):
        - Header: border-b border-gray-200 bg-white
          - Contenedor mx-auto max-w-3xl px-4 py-4
          - h1 "BugShot" text-xl font-semibold text-gray-900
        - Main: mx-auto max-w-3xl px-4 py-8
          - Empty state (id="empty-state"): flex col centrado, py-24
            - Icono grande (emoji microfono o SVG, text-6xl text-gray-300)
            - "No hay grabaciones" text-lg font-medium text-gray-600
            - "Pulsa un boton para empezar a grabar" text-sm text-gray-400
          - Feed container (id="feed"): div con class space-y-4, vacio
        - Botones flotantes: fixed bottom-6 left-1/2 -translate-x-1/2 flex gap-3
          - Boton "Audio": rounded-full bg-accent px-6 py-3 font-medium text-white shadow-lg hover:bg-accent-hover transition-colors
          - Boton "Video": mismo estilo que Audio
          - En movil (sm:): botones deben ser accesibles sin solapar contenido

   - Fondo general: bg-[#f5f5f5] en el body o div#app (fondo claro segun decision)

2. Crear src/js/app.js:
   - Funcion isBrowserCompatible():
     - Detectar 'SpeechRecognition' in window || 'webkitSpeechRecognition' in window
     - Detectar typeof MediaRecorder !== 'undefined'
     - Detectar navigator.mediaDevices?.getUserMedia
     - Retornar true solo si las 3 existen
   - Al cargar (DOMContentLoaded o top-level en module):
     - Si NO compatible: ocultar #app (display none), mostrar #unsupported (display flex)
     - Si compatible: no hacer nada (app visible, unsupported oculto)

3. Crear src/css/styles.css:
   - Estilos minimos complementarios a Tailwind:
     - body { margin: 0 } (reset basico)
     - Smooth scrolling: html { scroll-behavior: smooth }
     - Transicion hover para botones flotantes si no se puede con Tailwind utilities
   - No duplicar lo que Tailwind ya cubre
