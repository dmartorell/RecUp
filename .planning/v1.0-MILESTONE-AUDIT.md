## Deuda Técnica

### Phase 2: visualizer.js huérfano
- **Archivo:** `src/js/visualizer.js`
- **Issue:** El archivo existe pero no se importa desde ningún módulo. La lógica de waveform se reimplementó inline en `app.js` usando `analyser` directamente.
- **Impacto:** Dead code, sin efecto en runtime. No afecta ningún requirement.
- **Acción sugerida:** Eliminar el archivo.

### Phase 4: Detección de error parcial en adjuntos
- **Archivo:** `server/routes/attachment.js` L51 + `src/js/ticket-modal.js`
- **Issue:** `data.uploaded` es un array de resultados, no un número. La comprobación `data.uploaded > 0` siempre es `true` por coerción JS (`[] > 0` → `0 > 0` = false; `[{...}] > 0` → `1 > 0` = true). El botón "Reintentar adjuntos" aparece correctamente cuando hay uploads parciales, pero en un fallo total de multi-archivo podría no detectarse correctamente.
- **Impacto:** UX menor — en fallo total de multi-archivo, el botón de reintentar podría mostrarse cuando no debería. No hay crash.
- **Acción sugerida:** Cambiar a `data.uploaded.length > 0` en attachment.js y alinearlo con el cliente.

### Cross-cutting: Rutas API sin verificación de sesión
- **Rutas:** /api/summarize, /api/attachment
- **Issue:** No verifican el token de sesión. /api/ticket tiene guarda implícita (email debe ser miembro de workspace ClickUp), pero las otras dos rutas son accesibles sin sesión.
- **Impacto:** Aceptable para herramienta interna sin datos sensibles en servidor. A revisar si se expone externamente.
- **Acción sugerida:** Añadir middleware de sesión cuando se implemente INFR-01 (v2).
